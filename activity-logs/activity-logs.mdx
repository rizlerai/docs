## üîç Filtering and Searching Activities

The Activity Log provides powerful filtering options to help you find specific activities quickly. The filter section is located at the top of the Activity Log page.

** Filter Options**

<CardGroup cols={2}>
<Card title="üë• User Filter" icon="users">
**Purpose**: Filter activities by specific user

- Select a specific user from the dropdown
- Shows only activities performed by that user
- Helpful for reviewing actions by a specific team member
- **Default**: "All Users" (shows all activities)
</Card>

<Card title="üì¶ Type Filter" icon="boxes">
**Purpose**: Filter by module or feature

- Narrow down to specific modules or features
- Focus on changes in particular areas of the system
- Tracks activities across all available modules
- **Default**: "All Types" (shows all activity types)
</Card>

<Card title="‚ö° Event Filter" icon="bolt">
**Purpose**: Filter by action type

Available options:
- **Created** - New records added to the system
- **Updated** - Existing records modified
- **Deleted** - Records removed from the system
- **Transferred** - Records moved between entities

**Default**: "All Events" (shows all event types)
</Card>

<Card title="üìÖ Date Range Filter" icon="calendar">
**Purpose**: Filter by time period

Predefined options:
- **Today** - Today's activities only
- **Yesterday** - Previous day's activities
- **This Week** - Current week's activities
- **Last 7 Days** - Past week
- **Last 30 Days** - Past month
- **This Month** - Current month
- **Last Month** - Previous month
- **Custom dates** - Your own date range

**Default**: "Today"
</Card>
</CardGroup>

**üìÜ Custom Date Range**

<Steps>

<Step title="Set End Date">

<Step title="Apply Filter">

</Steps>

**üîò Applying and Resetting Filters**

<CardGroup cols={2}>
<Card title="‚úÖ Apply Filters" icon="filter">

<Card title="üîÑ Reset Filters" icon="arrow-rotate-left">
**How to Reset:**

1. Click the **Reset** button (blue button with refresh icon)

> üí° **Tip**: Use Reset when you want to start fresh with your filtering.

</CardGroup>

### üéØ Combining Filters

You can combine multiple filters together to create more specific searches:

**Example Combinations:**

- **User + Event**: Filter by a specific user **AND** a specific event type to see only records that user created
- **Date Range + Type**: Filter by a date range **AND** a specific type to see all activities in a module during a certain period
- **Search + Filters**: Use search **AND** filters together to find very specific activities

> üí° **Pro Tip**: Start with broader filters and narrow down as needed. This helps you find exactly what you're looking for without missing relevant results.

## üìù What Gets Tracked

The Activity Log system monitors and records activities across all modules of the platform. Whenever any significant action is performed on any record in the system, it gets automatically logged.

### üé¨ Types of Actions Tracked

<CardGroup cols={2}>
<Card title="‚ûï Record Creation" icon="plus-circle">
**What's Captured:**

When a new record is created, the activity log captures:
- All initial values that were set
- Complete configuration at creation time
- User who created the record
- Exact timestamp and IP address

**Use Case**: See exactly how a record was configured when first created.
</Card>

<Card title="‚úèÔ∏è Record Updates" icon="pencil">
**What's Captured:**

When a record is modified, the log captures:
- Previous values (before the change)
- New values (after the change)
- Only fields that were actually modified
- Complete before/after comparison

**Use Case**: Compare states and see exactly what changed.
</Card>

<Card title="üóëÔ∏è Record Deletion" icon="trash">
**What's Captured:**

When a record is deleted, the log preserves:
- Snapshot of all values before removal
- Complete record data for historical reference
- User who performed the deletion
- Timestamp and IP address

**Use Case**: Maintain historical records even after data is deleted.
</Card>

<Card title="‚≠ê Special Actions" icon="star">
**What's Captured:**

Beyond basic CRUD operations, tracks:
- Feature activation/deactivation
- Item assignments to users/teams
- Ownership transfers
- Status changes (active/inactive, etc.)
- Authentication events (login, logout, password changes)
- Module-specific operations

**Use Case**: Track all significant actions, not just data changes.
</Card>
</CardGroup>
