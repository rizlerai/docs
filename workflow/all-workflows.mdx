## 🤔 What is a workflow?

A workflow is an automation you define: **when something happens** (a trigger), **the system runs the steps you arranged** (actions), in order. You can organize workflows into folders, turn them on or off, and copy them when you need a similar setup.

- **All Workflows** shows every workflow. Choosing a folder narrows the list to workflows stored in that folder.
- You can **add folders** to group workflows by team, campaign, or any label you like.

- Each folder can be **renamed** or **removed**. Removing a folder does not delete the workflows inside; they are simply moved out of that folder.
- Folders can be **reordered** by dragging the handle next to the folder name.


### 📋 Main list (right side)

The table shows:

| What you see | What it means |
|--------------|----------------|
| **Name** | Opens the visual builder for that workflow when you click it. |
| **Status** | Whether the workflow is a draft, active, or turned off (see [Workflow status](#workflow-status) below). |
| **Created By** | Who created the workflow. |
| **Created At** | When it was created. |
| **Action menu** | Opens options for that row (see below). |

### ⚙️ Actions from the list

From the **⋯** menu on a row you can:

<img
  src="/images/workflow/edit-workflows.png"
  alt="Edit workflow classic form"
  className="rounded-lg border shadow-sm"
/> 

- **View Detail** — See workflow summary information and key details in a dedicated panel.
- **Edit** — Open the visual builder to change triggers, steps, and settings.
- **Change Folder** — Move the workflow to another folder or to no folder.
- **Activate** or **Deactivate** — Turn the workflow on or off without deleting it.
- **Duplicate** — Create a copy named with “(Copy)”, saved as a draft, and open it in the builder.
- **Delete** — Remove the workflow permanently (you will be asked to confirm).

Use **Create Workflow** to start a new workflow. A new item is created for you and you are taken straight into the builder.

## 🔄 Workflow status

| Status | Meaning |
|--------|---------|
| **Draft** | Still being designed. It will not run until you publish it. |
| **Active** | Published and running: it can run when its triggers occur. |
| **Inactive** | Turned off: it will not run until you activate it again. |

## 📝 Create Workflow (classic form)

There is a screen where you can type a **name**, optional **description**, choose a **trigger type** from a fixed list (for example new contact, form submission, appointment booked), and optionally fill **extra fields** that depend on the type you picked. New workflows from this path start as **drafts**.

> **Note:** The usual path today is **Create Workflow** from **All Workflows**, which creates a draft and sends you to the **builder** immediately. The classic form is an alternative if your team still uses it.



## ✏️ Edit Workflow (classic form)

This screen lets you change the **name**, **description**, **trigger type**, and **status** using the same kind of trigger list and optional extra fields as the create form. It also explains **Draft**, **Active**, and **Inactive** in plain language and shows who created the workflow and when.

For full visual design (triggers on the canvas, action steps, filters), use **Edit** from the list to open the **builder**.

## 🏗️ Workflow Builder

The builder is where you design the automation visually. It opens in a full-screen layout focused on your workflow.

<img
  src="/images/workflow/workflow-builder.png"
  alt="Workflow builder canvas"
  className="rounded-lg border shadow-sm"
/>

### ⬆️ Top bar

- **Workflow name** — Shown at the top; you can edit it with the small edit control beside it.
- **Builder** — The design surface for triggers and steps.
- **Settings** — Time zone, who may run the workflow, time windows, and default sender details for messages (see [Settings (inside the builder)](#settings-inside-the-builder) below).
- **Zoom** — Make the canvas larger or smaller.
- **SOPs** — Opens help or standard procedures for using workflows, if your team has configured them.
- **Test** — Reserved for testing the workflow; behavior depends on product updates.
- **Publish / Draft** — Toggles whether the workflow is live or stays a draft.
- **Save** — Saves your design. The system may also save your work in the background as you go.
- **Back** — Returns to **All Workflows**.

### 🎨 Builder canvas

<img
  src="/images/workflow/edit-workflows-details.png"
  alt="Workflow builder with trigger and steps"
  className="rounded-lg border shadow-sm"
/>

1. **Add New Trigger** — Opens a panel of available triggers, grouped by category (Contacts, Appointments, Opportunities, Events). You can search. Pick one to add it to your design.

<img
  src="/images/workflow/add-triggers.png"
  alt="Choosing a workflow trigger"
  className="rounded-lg border shadow-sm"
/>

2. **Trigger cards** — Each trigger has a name and description you can set. Some triggers allow **filters** so the workflow only runs when extra conditions match (for example certain tags, sources, or call details — depending on the trigger).
3. **Steps under a trigger** — After a trigger, you add **actions** one after another (for example send a text, wait, update tags). Steps run **top to bottom** in the order you place them.

<img
  src="/images/workflow/add-actions.png"
  alt="Adding workflow actions"
  className="rounded-lg border shadow-sm"
/>

4. **Connectors** — Lines show how the trigger connects to its steps.
5. **Mini map** — A small overview of a large workflow; appears when useful.
6. **Selecting a step** — When you select a trigger or action, a small toolbar may offer **Edit**, **Delete**, or **Duplicate** for that piece.

Triggers and actions are chosen from lists maintained for your organization. Typical **trigger** groups include:

- **Contacts** — Contact changed, contact created, tag added or removed, disposition change, note added or changed.
- **Appointments** — Appointment status changes, customer booked an appointment.
- **Opportunities** — Pipeline stage change.
- **Events** — Dialer call details, AI call details, incoming SMS (sometimes with extra choices), or a manual trigger.

Typical **actions** include working with contact tags and disposition, notes, assigned users, deleting a contact (destructive — use with care), sending information to another system by link, sending SMS, **waiting** for a delay or until a time window, and moving contacts to a pipeline stage.

Each action opens its own **configuration** when you edit it (what to send, which tag, which pipeline stage, and so on).

### ⚙️ Settings 

**Timezone**

- Chooses which timezone is used for **Wait** steps and for **time window** rules.

**Allowed users** (on accounts that show this section)

- Optionally limit which team members may start or trigger this workflow. If nobody is selected, all users in the account are allowed.

**Time window**

- Optional. When enabled, you set a **start time**, **end time**, and **days of the week**. Outside that window, automated actions wait until the next allowed time. Help text on the screen explains how this interacts with Wait steps.

**Sender details**

- Default **from name**, **from email**, and **from number** for outgoing messages where the product uses these defaults. Individual message actions may still override them where the product allows.

Use **Save Settings** to store changes on this tab.

![Workflow Settings](/images/workflow/workflow-settings.png)



## 💾 Saving your work

- Use **Save** on the builder when you want to be sure everything is stored.
- The builder may **save automatically** as you work; you can still use **Save** whenever you prefer.

Your design — which triggers you added, their filters, and the ordered list of actions — is kept together with the workflow’s name, description, and status.
