# Webhooks

Configure webhooks to automatically send data from your AI Sync system to external endpoints, enabling seamless integration with CRM systems, marketing tools, and custom applications.
![Dashboard Analytics Overview](/images/setup/webhooks/overview.png)

## Overview

Webhooks are HTTP callbacks that send real-time data from AI Sync to external systems when specific events occur. They enable automated data synchronization, lead management, and integration with your existing business tools.

## Endpoint Configuration

### HTTP Method

Webhooks use **POST** method to send data to external endpoints:

```http
POST https://your-endpoint.com/webhook
Content-Type: application/json
X-Webhook-Signature: sha256=abc123...

{
  "event": "contact.created",
  "data": { ... }
}
```

### URL Structure

Webhook URLs follow a secure pattern with encoded parameters:

```
https://app.aisync.link.com/webhooks/contact/eyJpdil6lkM3QnhCS3F1eHp5Mk1XcGhZZEErc1E9PSIsInZhbHVlljoiaTJJZ09YSi9Nd103SFJla0pmcU1Udz09liwibWFjIjoiZWVjZWViM2E4NDdjYzAzYTBkYjViMzAyY2M5NjczMmYzZmUyOTI5MTg1NDNjZjE1MDQyMDQ1YjYxYzIkNTBINilsInRhZyl6liJ9
```

### Security Features

- **Encoded Parameters**: All sensitive data is encrypted in the URL
- **Signature Verification**: HMAC signatures ensure data integrity
- **Rate Limiting**: Built-in protection against abuse
- **IP Whitelisting**: Optional IP address restrictions

## Integration Examples

### CRM Integration

<Image
  src="/images/webhook-integration-crm.png"
  alt="CRM Integration Flow - Diagram showing data flow from AI Sync webhook to CRM system"
  width={700}
  height={300}
  caption="CRM Integration - Seamlessly sync contact data with your CRM"
/>

```javascript
// Example: Salesforce webhook handler
app.post('/webhook/rizler', (req, res) => {
  const { event, data } = req.body;
  
  if (event === 'contact.created') {
    // Create new lead in Salesforce
    salesforce.createLead({
      FirstName: data.first_name,
      LastName: data.last_name,
      Email: data.email,
      Phone: data.phone_number,
      LeadSource: 'AI Sync Webhook'
    });
  }
  
  res.status(200).json({ success: true });
});
```

### Marketing Automation

```javascript
// Example: Mailchimp integration
app.post('/webhook/marketing', (req, res) => {
  const { data } = req.body;
  
  // Add contact to email list
  mailchimp.lists.addListMember('audience-id', {
    email_address: data.email,
    status: 'subscribed',
    merge_fields: {
      FNAME: data.first_name,
      LNAME: data.last_name,
      PHONE: data.phone_number
    }
  });
  
  res.status(200).json({ success: true });
});
```

### Custom Application

```javascript
// Example: Custom dashboard integration
app.post('/webhook/dashboard', (req, res) => {
  const { event, data } = req.body;
  
  // Update real-time dashboard
  io.emit('newContact', {
    event: event,
    contact: data,
    timestamp: new Date()
  });
  
  // Store in database
  db.contacts.insert(data);
  
  res.status(200).json({ success: true });
});
```

## Best Practices

### Security

<Image
  src="/images/webhook-security.png"
  alt="Webhook Security Features - Diagram showing HTTPS, signatures, rate limiting, and IP whitelisting"
  width={700}
  height={300}
  caption="Webhook Security - Protect your data with multiple security layers"
/>

- **Use HTTPS**: Always use secure connections for webhook endpoints
- **Verify Signatures**: Implement signature verification on your endpoints
- **Rate Limiting**: Implement rate limiting to prevent abuse
- **IP Whitelisting**: Restrict webhook sources to known IP addresses

### Reliability

- **Idempotency**: Design endpoints to handle duplicate webhooks safely
- **Error Handling**: Implement proper error handling and logging
- **Retry Logic**: Configure appropriate retry intervals and limits
- **Monitoring**: Set up alerts for webhook failures

### Performance

- **Async Processing**: Process webhooks asynchronously when possible
- **Response Time**: Respond quickly to webhook requests (under 5 seconds)
- **Data Validation**: Validate incoming data before processing
- **Resource Management**: Monitor endpoint resource usage

## Advanced Configuration

### Custom Headers

Add custom headers to webhook requests for authentication or identification:

```http
X-API-Key: your-api-key
X-Webhook-ID: wh_12345
X-Event-Type: contact.created
User-Agent: AI-Sync-Webhook/1.0
```

### Conditional Webhooks

Configure webhooks to trigger only under specific conditions:

- **Field Values**: Trigger based on specific field values
- **Time Windows**: Only send webhooks during business hours
- **Data Volume**: Limit webhooks based on data volume thresholds
- **User Permissions**: Restrict webhooks based on user roles

### Webhook Chaining

Create sequences of webhooks that trigger in order:

```json
  "id": "wh_12345",
  "title": "Contact Sync",
  "url": "https://api.example.com/webhook",
  "method": "POST",
  "powerlist_id": "pl_67890",

  "is_active": true,

  "updated_at": "2025-01-06T22:40:00Z",
  "hits": 8,
  "last_hit": "2025-08-22T19:44:00Z"
}
```

### Response Codes

- **200**: Webhook delivered successfully
- **400**: Bad request (invalid payload)
- **401**: Unauthorized (authentication failed)
- **403**: Forbidden (access denied)
- **404**: Endpoint not found
- **500**: Server error (endpoint error)
- **Timeout**: Endpoint didn't respond within time limit
