The **Contacts** page is the main place to view, search, filter, and manage contacts in RizzDial. It gives you a modern table view for daily contact management and quick access to each contact's activity history.


<img
  src="/images/manage/contact/overview.png"
  alt="Contacts page with search, filters, and contact table"
  className="rounded-lg border shadow-sm"
/>

## ✨ What you can do

- View all visible contacts.
- Search contacts by name, phone number, or email.
- Filter contacts by tag.
- Filter contacts by powerlist.
- Sort contacts by newest, oldest, or name.
- Open a contact's activity page to review calls, messages, notes, and workflows.
- Select contacts and apply bulk actions.
- Import new contacts from a CSV file.

## 📋 Contact table

The main table shows the most important contact details:

| Column | What it shows |
|--------|---------------|
| **Name** | Contact name |
| **Phone** | Phone number |
| **Email** | Email address |
| **Tags** | Tags assigned to the contact |
| **Added** | Date the contact was added |

Each row is clickable, or use the external-link icon on a row to open that contact's **activity page**. See [Contact activity page](#-contact-activity-page) below for what you can do there.

**🔍 Search**

Use the search bar at the top to find contacts.

The search supports:

- First name
- Last name
- Email
- Phone number

Phone search works best when you enter at least four digits.

The search input is debounced, so the page waits briefly while you type before reloading results. This keeps the page responsive and avoids unnecessary API calls.

**🏷️ Tag filter**

Filter the list by a contact tag using the **Tags** dropdown.

The tag dropdown shows available tags for your company. When possible, it also shows the number of visible contacts using each tag.

You can also click a tag chip inside the contact table to immediately filter by that tag.

**📂 Powerlist filter**

Filter contacts by powerlist using the **Powerlist** dropdown.

The powerlist picker shows available powerlists and their contact counts. Selecting a powerlist limits the table to contacts inside that list.

This is helpful when you want to work on one campaign list or one imported segment at a time.

**↔️ Sorting**

Sort contacts using the dropdown next to the search bar:

| Sort option | What it does |
|-------------|--------------|
| **Newest first** | Shows recently added contacts at the top |
| **Oldest first** | Shows oldest contacts at the top |
| **Name A-Z** | Sorts contacts alphabetically by name |

Sorting makes it easier to review fresh imports, older records, or alphabetized contact lists.

**📄 Pagination**

The page loads contacts in pages instead of loading the whole database at once.

By default, it requests 50 contacts per page. This keeps the page fast, even for companies with large contact tables.

Use **Prev** and **Next** to move between pages.

**📥 Import CSV shortcut**

The **Import CSV** button sends you to the contact import page.

This is useful when the contact list is empty or when you need to add a new batch of leads.

**🔄 Refresh button**

The refresh button reloads the current list using the active search, filters, sort, and page.

Use it after imports or bulk actions to see the latest data.

## 📜 Contact activity page

Click a contact row or the external-link icon on a row to open the **Contact activity page**. This is the full history view for one contact — not the main Contacts table.

<img
  src="/images/manage/contact/activity.png"
  alt="Contact activity page with details, notes, and call timeline"
  className="rounded-lg border shadow-sm"
/>

### 🔙 Header and navigation

At the top of the activity page you will see:

- **Back** — return to the Contacts list.
- **Contact ID and phone** — for example `Contact #18383` and the primary phone number.
- **Dashboard** and **Contacts** — quick links to other areas of the app.
- **Refresh** — reload the activity timeline and contact details.

### 🧾 Contact details

The **Contact details** section summarizes the contact record:

| Field | What it shows |
|-------|---------------|
| **Summary cards** | Counts for Messages, Calls, Emails, Notes, and Workflows |
| **Primary phone** | Main phone number for the contact |
| **Added** | Date and time the contact was created |
| **Powerlist** | Powerlist the contact belongs to |
| **Consent** | Consent status — for example **No consent on file** when consent has not been recorded |

If the contact has no name saved, the page may show **Unnamed contact**.

### 🗂️ Activity filters

Use the filter tabs to narrow the timeline:

| Tab | What it shows |
|-----|---------------|
| **All** | Every activity type |
| **Messages** | SMS and chat messages |
| **Calls** | Voice call logs |
| **Email** | Email activity |
| **Workflows** | Workflow runs tied to this contact |

Each tab shows a count so you can see how many items exist in each category.

### 📝 Internal team note

The **Internal team note** box lets your team leave a quick note that is only visible inside RizzDial — not sent to the contact.

- Type your note in the text area.
- Press **Cmd-Enter** (Mac) or use **Post note** to save.

Use this for handoffs, context, or reminders about the contact.

### 📅 Activity timeline

Below the details and note area, the **timeline** lists interactions in order, grouped by date (for example **THURSDAY 11 JUNE**).

Each entry shows what happened and when. For **calls**, a typical entry includes:

- Badges such as **CALL**, **Outbound**, and **Completed**
- Time of the call
- A transcript preview with speaker labels
- **From** and **To** phone numbers
- A **Recording** link to play the call audio when a recording is available

Messages, emails, and workflow events appear in the same timeline when you use the **All** filter or the matching tab.

## ⚡ Bulk actions

Select one or more contacts from the table. After contacts are selected, a bulk action bar appears.

<img
  src="/images/manage/contact/bulk-actions.png"
  alt="Bulk action bar with Tag and Delete options on selected contacts"
  className="rounded-lg border shadow-sm"
/>

Available actions:

**🏷️ Apply tag**

When you apply a tag, the selected tag is added to all selected contacts.

If workflow automations are connected to tag changes, applying a tag can trigger those workflows.

**➖ Remove tag**

When you remove a tag, that tag is removed from all selected contacts that currently have it.

If workflow automations are connected to tag removal, removing a tag can trigger those workflows.

**🗑️ Delete contacts**

The delete action asks for confirmation before running.

When possible, contacts are soft deleted by marking them as deleted. This hides them from the Contacts page while keeping their activity history available. If your contact table does not support soft delete, contacts are removed directly.

**👁️ Visibility and permissions**

The Contacts page only shows visible contacts.

If a contact is marked as deleted, it is hidden from the list.

For users who are not company-level users and do not have permission to view all contacts, results may be restricted to contacts inside assigned powerlists only.
