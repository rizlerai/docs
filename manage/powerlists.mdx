This guide explains **PowerLists** in simple language: purpose, main list, actions, setup form, and upload flow.

A **PowerList** is a named calling list for outbound work.

For each list, you can:

- set a unique list name,
- define dialing behavior,
- assign members and tags,
- connect optional CRM location/tag settings,
- define call-outcome actions (dispositions),
- and upload contacts from files.


## 🖥️ Main screen (list view)

**🏷️ Title and trail**

Top area usually shows **PowerLists** and a breadcrumb like:

`PowerList -> List`

**🏁 Actions at the top**

- **New PowerList**: starts list creation.
- **SOPs**: opens built-in standard operating procedure reference.

**🗂️ Filters (above table)**

- **Tags**: filter lists by one or more assigned tags.
- **PowerList Member**: filter lists by assigned member(s), when field is available.
- **Filter**: applies selected filter values.

**⚙️ Table controls**

- Rows-per-page selector.
- Search box to narrow visible rows.


## 🗃️ Columns in the main table

| What you see | What it means |
|--------------|----------------|
| **#ID** | Row number in current view. |
| **Active** | Row menu entry point (`...`) with list actions. |
| **ID** | System identifier for support/export matching. |
| **PowerList** | List name; tag chips may appear next to it. |
| **# Uploaded** | Count of contacts uploaded to list. |
| **# Dialed** | Total contacts dialed over time. |
| **# Daily Dialed** | Contacts dialed during current day. |
| **# Removed** | Contacts removed from this list. |
| **Created At** | Creation date/time in your timezone. |

## ⚡ Menu actions (under Active)

Action availability depends on role and list type. If visible, they usually mean:

- **Status**: turn list on/off for dialing.
- **Duplicate**: clones setup into a new list name; contacts are not copied.
- **Edit**: opens full configuration form.
- **View**: opens contacts view for that list.
- **Delete**: permanently removes list and related contacts after confirmation.
- **Download Statistics**: exports metrics and outcome summaries.
- **Export Contacts**: spreadsheet-friendly contacts export.
- **CSV Contacts**: CSV export format.
- **PDF Contacts**: report-style PDF export.
- **Reset**: resets dialing state of contacts to fresh/new without removing contacts.

## 📝 Create or edit a PowerList 

Use **New PowerList** or **Edit** from row menu.

Use **Back** to leave without saving, and **Save** to apply changes.

<img
  src="/images/manage/powerlists/add-new.png"
  alt="New PowerList setup form"
  className="rounded-lg border shadow-sm"
/>

Many labels include help icons with short guidance on hover/focus.

### 👤 People and naming

- **PowerList Member**: choose one or more members when field is shown.
- **PowerList Name**: unique, clear name for your company.
- **Max Contacts**: list capacity limit; system may enforce minimum.

### 📞 Calling behavior

- **Must Disposition Calls**: enforce outcome selection before moving on.
- **Seconds before Timeout**: ring wait before unanswered handling.
- **Dial at a Time**: concurrent outbound calls for list.
- **Max Attempts Daily**: daily try limit per contact.
- **Max Attempts Total**: total try limit per contact.
- **Next PowerList**: optional next list handoff target.

### 🏷️ Organization and follow-up

- **Tags**: labels for grouping/filtering lists later.
- **AI Agent For Callback**: optional callback assistant assignment.
- **Dialed Reset Daily**: resets daily counters at day boundary.
- **Cool Down Time**: wait period before retrying same contact.

### 🔗 CRM connection (GHL)

- **GHL Location**: destination location for posting actions.
- **GHL Tag On Disconnected**: optional tag applied for disconnected outcomes.

### 📝 Contact fields (Collect User Information)

Choose which data points are stored for contacts in this list (name/email/address/custom fields etc., based on your account).

### 🏁 Dispositions

For each outcome label, choose one action:

- **No Action**
- **Remove From List**
- **Post To GHL**
- **Post To GHL & Never Call**

Choose actions based on your team workflow and compliance logic.

## ⬆️ Uploading contacts

### 🔁 Duplicate handling mode

- **Ignore**: skip duplicate rows.
- **Overwrite**: replace existing matching data.
- **Merge**: combine incoming and existing data.

### 📂 File selection

- Upload supported spreadsheet/CSV file types shown in UI.
- Then open **Mapping Tool**.

### 🗺️ Mapping rules

- Map file columns to internal fields.
- At least one mapped **Phone Number** field is required.
- Unused columns can be mapped to **Ignore**.

### 💾 Save and progress

Click **Save** to store list settings and start import (if mapping is complete).

Progress area may show:

- job/reference id,
- current status,
- result summary.

Use **Check Status** to refresh progress, **Refresh Table** to update list counts, and **Close** when done.
